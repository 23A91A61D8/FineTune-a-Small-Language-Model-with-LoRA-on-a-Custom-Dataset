## Project Overview

This project demonstrates **parameter-efficient fine-tuning (PEFT)** of a small causal language model using **LoRA (Low-Rank Adaptation)**. The model is trained on a custom dataset of **informal-to-formal English conversion** pairs, teaching it to rewrite casual messages into professional business language.

### Task

> **Informal → Formal English Conversion**  
> *Input:* `"hey can u send the report asap its kinda urgent lol"`  
> *Output:* `"Could you please send me the report at your earliest convenience? This matter is quite urgent."`

## Quick Start

### Option A — Local (Git Bash / Terminal)

#### Prerequisites
- Python 3.10+
- Git Bash (Windows) or any terminal
- *(Optional)* NVIDIA GPU with CUDA 12+

```bash
# 1. Clone the repository
git clone https://github.com/23A91A61D8/FineTune-a-Small-Language-Model-with-LoRA-on-a-Custom-Dataset.git
cd FineTune-a-Small-Language-Model-with-LoRA-on-a-Custom-Dataset

# 2. Create & activate a virtual environment
python -m venv venv
source venv/Scripts/activate      # Git Bash on Windows
# source venv/bin/activate         # macOS / Linux

# 3. Install dependencies
pip install -r requirements.txt

# 4. Set up environment variables
cp .env.example .env
# Edit .env if you want to change the model or hyper-parameters

# 5. Generate the dataset (creates dataset/training_data.jsonl)
python generate_dataset.py

# 6. Fine-tune the model (creates lora_adapter/ and output/loss_curve.png)
python train.py

# 7. Run inference (creates output/comparison.md)
python predict.py
```

### Option B — Docker (recommended for reproducibility)

```bash
# 1. Clone the repo
git clone https://github.com/23A91A61D8/FineTune-a-Small-Language-Model-with-LoRA-on-a-Custom-Dataset.git
cd FineTune-a-Small-Language-Model-with-LoRA-on-a-Custom-Dataset

cp .env.example .env

# 3. Run the full pipeline (generate → train → predict)
docker compose run --rm full

docker compose run --rm train      # generate dataset + train
docker compose run --rm predict    # inference only
```

> **GPU support (Docker):** Uncomment the `deploy:` block in `docker-compose.yml` and ensure `nvidia-container-toolkit` is installed on your host.

## Configuration

All configuration is done via the `.env` file (copy from `.env.example`):

| Variable | Default | Description |
|---|---|---|
| `MODEL_NAME` | `TinyLlama/TinyLlama-1.1B-Chat-v1.0` | Base model from HuggingFace |
| `DATASET_PATH` | `dataset/training_data.jsonl` | Training data location |
| `ADAPTER_DIR` | `lora_adapter` | Where to save LoRA weights |
| `OUTPUT_DIR` | `output` | Where to save PNG and Markdown |
| `EPOCHS` | `3` | Number of training epochs |
| `BATCH_SIZE` | `2` | Per-device batch size |
| `LEARNING_RATE` | `2e-4` | AdamW learning rate |
| `LORA_RANK` | `16` | LoRA rank (r) |
| `LORA_ALPHA` | `32` | LoRA scaling factor |
| `MAX_SEQ_LENGTH` | `512` | Max token length |
| `MAX_NEW_TOKENS` | `150` | Max tokens to generate |

### Using a larger model

To use the model specified in the task brief, update `.env`:


## How LoRA Works

```
 Frozen Base Model          LoRA Adapter
 ──────────────────         ────────────────────────
  Weight Matrix W            Matrix A  ×  Matrix B
  (4096 × 4096)              (4096×16)    (16×4096)
  NEVER updated              trained      trained

  Effective weight = W + ΔW
  where ΔW = A · B  (low-rank approximation)
```

- **Matrix B is initialised to zeros** → ΔW = 0 at start → stable training
- Only **~0.1% of parameters** are trained, saving ~10× GPU memory
- After training, adapter is **merged** into base weights at inference time

## Dataset

- **Format:** JSON Lines (`.jsonl`) — one JSON object per line
- **Size:** 100 instruction-response pairs
- **Task:** Informal-to-Formal English rewriting
- **Schema:**

```json
{
  "instruction": "Rewrite this message in a formal tone.",
  "input": "hey can u send the report asap",
  "output": "Could you please send me the report at your earliest convenience?"
}
```

## Training Details

| Setting | Value |
|---|---|
| Base model | TinyLlama-1.1B-Chat-v1.0 |
| LoRA rank (r) | 16 |
| LoRA alpha | 32 |
| Target modules | `q_proj`, `v_proj` |
| LoRA dropout | 0.05 |
| Epochs | 3 |
| Batch size | 2 (+ grad accum ×4 = effective 8) |
| Learning rate | 2e-4 (AdamW, warmup 5%) |
| Precision | fp16 (GPU) / fp32 (CPU) |

## Reflection

### What Improved
The fine-tuned model reliably eliminates contractions, informal abbreviations, and slang. It adopts appropriate formal salutations and closing phrases that are absent from the base model's outputs.

### What Didn't Improve
For very short inputs the fine-tuned model occasionally over-generates additional sentences not warranted by the original message. The base model handles some simple one-word replacements acceptably.

### Future Improvements
1. Expand dataset to **500+ examples** with more linguistic variety.
2. Add a **validation split** and implement early-stopping on validation loss.
3. Experiment with **higher LoRA rank** (r=32 or r=64) for richer adaptation.
4. Target additional modules (`k_proj`, `o_proj`) in the LoRA config.
5. Evaluate objectively using **BLEU / ROUGE / BERTScore** metrics.
6. Try **4-bit quantisation (QLoRA)** to reduce memory footprint further.
